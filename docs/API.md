# API Documentation

Full interactive documentation is auto-generated by FastAPI and available at:

- **Swagger UI**: `/docs`
- **ReDoc**: `/redoc`
- **OpenAPI JSON schema**: `/openapi.json`

This document summarizes the available endpoints.

## Authentication

All `/api/*` endpoints (except registration and login) require a Bearer JWT:

```
Authorization: Bearer <access_token>
```

| Method | Path | Description | Auth |
|---|---|---|---|
| POST | `/api/auth/register` | Register a new user (default role: viewer) | none |
| POST | `/api/auth/login` | Login (form-encoded `username`, `password`), returns JWT | none |

## Assets

| Method | Path | Description | Auth |
|---|---|---|---|
| GET | `/api/assets` | List all assets | any authenticated user |
| POST | `/api/assets` | Create a new asset (`name`, `asset_type_id`, `description?`, `tags?`) | any authenticated user |
| GET | `/api/assets/{id}` | Get a single asset | any authenticated user |
| DELETE | `/api/assets/{id}` | Delete an asset (cascades to its scan history) | any authenticated user |
| GET | `/api/asset-types` | List available asset types (domain/ip/url) | any authenticated user |

## Scans

| Method | Path | Description | Auth |
|---|---|---|---|
| POST | `/api/scans` | Run a full scan for `asset_id` (synchronous; returns when complete) | any authenticated user |
| GET | `/api/scans` | List recent scan results | any authenticated user |
| GET | `/api/scans/{id}` | Get a single scan result | any authenticated user |

## Alerts

| Method | Path | Description | Auth |
|---|---|---|---|
| GET | `/api/alerts` | List alerts, newest first | any authenticated user |
| PATCH | `/api/alerts/{id}` | Update `is_resolved` | any authenticated user |

## Reports

| Method | Path | Description | Auth |
|---|---|---|---|
| POST | `/api/reports` | Generate a report (`asset_id?`, `report_type`: `pdf`\|`csv`\|`xlsx`) | any authenticated user |
| GET | `/api/reports` | List generated reports | any authenticated user |

Report files are written to `generated_reports/` and can be downloaded via
the HTML route `GET /reports/{id}/download` (returns the file directly).

## Users (admin only)

| Method | Path | Description | Auth |
|---|---|---|---|
| GET | `/api/users` | List all users | admin |
| POST | `/api/users` | Create a user with a specific role | admin |
| PATCH | `/api/users/{id}` | Update a user (email, full_name, is_active, role_id) | admin |

## System

| Method | Path | Description | Auth |
|---|---|---|---|
| GET | `/api/health` | Health check (used by Docker healthcheck / Prometheus) | none |
| GET | `/api/dashboard/stats` | Aggregate counts (total assets, open/critical alerts) | any authenticated user |
| GET | `/api/audit-logs` | List audit log entries | admin |

## Example: full workflow via curl

```bash
# 1. Register
TOKEN=$(curl -s -X POST http://localhost:8000/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"username":"alice","email":"alice@example.com","password":"SecurePass123"}' \
  | python3 -c "import sys,json;print(json.load(sys.stdin)['access_token'])")

# 2. Create an asset
curl -s -X POST http://localhost:8000/api/assets \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d '{"name":"example.com","asset_type_id":1}'

# 3. Run a scan
curl -s -X POST http://localhost:8000/api/scans \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d '{"asset_id":1,"scan_type":"full"}'

# 4. Generate a PDF report
curl -s -X POST http://localhost:8000/api/reports \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d '{"asset_id":1,"report_type":"pdf"}'
```
